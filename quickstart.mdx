This guide walks you through installing TypeScript, writing a typed source file, compiling it, and running the output. The entire process takes under five minutes.

<Steps>

<Step title="Install TypeScript">

Install TypeScript as a development dependency in your project:

<CodeGroup>

```bash npm
mkdir hello-typescript && cd hello-typescript
npm init -y
npm install --save-dev typescript
```

```bash yarn
mkdir hello-typescript && cd hello-typescript
yarn init -y
yarn add --dev typescript
```

```bash pnpm
mkdir hello-typescript && cd hello-typescript
pnpm init
pnpm add --save-dev typescript
```

</CodeGroup>

Verify the installation:

```bash
npx tsc --version
# Version 6.0.0
```

</Step>

<Step title="Create a tsconfig.json">

Generate a `tsconfig.json` at the root of your project:

```bash
npx tsc --init
```

Replace the generated file with this configuration tailored for a Node.js project:

```json tsconfig.json
{
  "compilerOptions": {
    "target": "es2022",
    "module": "commonjs",
    "moduleResolution": "node",
    "outDir": "./dist",
    "rootDir": "./src",
    "strict": true,
    "esModuleInterop": true,
    "forceConsistentCasingInFileNames": true,
    "skipLibCheck": true
  },
  "include": ["src"]
}
```

What each option does:

- **`target: "es2022"`** вЂ” Emit JavaScript targeting ES2022, which Node.js 18+ supports natively.
- **`module: "commonjs"`** вЂ” Use `require`/`module.exports` for the emitted modules.
- **`outDir: "./dist"`** вЂ” Write compiled `.js` files into a `dist/` directory.
- **`rootDir: "./src"`** вЂ” TypeScript source files live in `src/`.
- **`strict: true`** вЂ” Enable the full suite of strict checks: `noImplicitAny`, `strictNullChecks`, `strictFunctionTypes`, `strictBindCallApply`, `strictPropertyInitialization`, `noImplicitThis`, and `useUnknownInCatchVariables`.
- **`esModuleInterop: true`** вЂ” Allow default imports from CommonJS modules.

</Step>

<Step title="Write a TypeScript source file">

Create the source directory and your first TypeScript file:

```bash
mkdir src
```

```typescript twoslash src/index.ts
interface User {
  id: number;
  name: string;
  email: string;
  role: "admin" | "editor" | "viewer";
}

function formatUser(user: User): string {
  return `[${user.role.toUpperCase()}] ${user.name} <${user.email}>`;
}

function getAdmins(users: User[]): User[] {
  return users.filter((u) => u.role === "admin");
}

const users: User[] = [
  { id: 1, name: "Alice", email: "alice@example.com", role: "admin" },
  { id: 2, name: "Bob", email: "bob@example.com", role: "editor" },
  { id: 3, name: "Carol", email: "carol@example.com", role: "viewer" },
];

const admins = getAdmins(users);
console.log(`Admins (${admins.length}):`);
admins.forEach((u) => console.log(" ", formatUser(u)));
```

This example demonstrates several TypeScript features that plain JavaScript lacks:

- An `interface` that describes the exact shape of a user object
- A union type (`"admin" | "editor" | "viewer"`) that restricts `role` to known string values
- Explicit return types on functions so the compiler validates your implementation
- Type-safe array operations вЂ” `filter` returns `User[]`, not `any[]`

Try introducing a type error to see the compiler catch it:

```typescript twoslash
// @errors: 2322
interface User {
  id: number;
  name: string;
  email: string;
  role: "admin" | "editor" | "viewer";
}

const user: User = {
  id: 1,
  name: "Alice",
  email: "alice@example.com",
  role: "superuser", // Error: Type '"superuser"' is not assignable to type '"admin" | "editor" | "viewer"'
};
```

</Step>

<Step title="Compile with tsc">

Run the TypeScript compiler from your project root:

```bash
npx tsc
```

If there are no errors, the compiler exits silently and writes the compiled output to `dist/`:

```

```

Inspect the emitted JavaScript to see what TypeScript produced:

```javascript dist/index.js
"use strict";
function formatUser(user) {
    return `[${user.role.toUpperCase()}] ${user.name} <${user.email}>`;
}
function getAdmins(users) {
    return users.filter((u) => u.role === "admin");
}
const users = [
    { id: 1, name: "Alice", email: "alice@example.com", role: "admin" },
    { id: 2, name: "Bob", email: "bob@example.com", role: "editor" },
    { id: 3, name: "Carol", email: "carol@example.com", role: "viewer" },
];
const admins = getAdmins(users);
console.log(`Admins (${admins.length}):`);
admins.forEach((u) => console.log(" ", formatUser(u)));
```

All type annotations are erased вЂ” the output is plain JavaScript with no runtime dependency on TypeScript.

<Tip>
Pass `--watch` (or `-w`) to recompile automatically whenever source files change:

```bash
npx tsc --watch
```
</Tip>

</Step>

<Step title="Run the compiled output">

Run the emitted JavaScript with Node.js:

```bash
node dist/index.js
```

You should see:

```
Admins (1):
  [ADMIN] Alice <alice@example.com>
```

</Step>

<Step title="Run TypeScript directly with ts-node (optional)">

For development, `ts-node` lets you run TypeScript files without a separate compile step:

<CodeGroup>

```bash npm
npm install --save-dev ts-node
```

```bash yarn
yarn add --dev ts-node
```

```bash pnpm
pnpm add --save-dev ts-node
```

</CodeGroup>

Run your file directly:

```bash
npx ts-node src/index.ts
```

`ts-node` compiles TypeScript in memory and executes it immediately вЂ” no `dist/` directory required.

<Note>
`ts-node` is useful for scripts and development workflows. For production, always pre-compile with `tsc` and ship the emitted JavaScript.
</Note>

For ESM projects or for faster startup, consider `tsx` as an alternative:

```bash
npx tsx src/index.ts
```

</Step>

</Steps>
