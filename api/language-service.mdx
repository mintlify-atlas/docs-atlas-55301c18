The TypeScript Language Service provides the full set of editor features that power IDEs: completions, diagnostics, go-to-definition, find references, refactoring, and more. It operates on a set of in-memory file snapshots managed by a **host** you implement, making it suitable for editors, linters, and any tooling that needs rich semantic information without running `tsc`.

## How it works

You supply a `LanguageServiceHost` that answers questions about your project (which files exist, what their contents are, what compiler options to use). The language service builds a `Program` from that information and exposes semantic operations on top of it. Because the host owns file state, you can feed it content from an editor buffer rather than the filesystem.

<Steps>

<Step title="Implement LanguageServiceHost">

Create an object that satisfies the `LanguageServiceHost` interface. At minimum you need to implement:
- `getCompilationSettings` — compiler options
- `getScriptFileNames` — list of files in the project
- `getScriptVersion` — version string (bump this when content changes)
- `getScriptSnapshot` — the file content as an `IScriptSnapshot`
- `getCurrentDirectory` — working directory
- `getDefaultLibFileName` — path to `lib.d.ts`
- `readFile` — read arbitrary files from disk
- `fileExists` — check whether a file exists

</Step>

<Step title="Call createLanguageService">

Pass your host to `createLanguageService` to get back a `LanguageService` instance.

</Step>

<Step title="Keep versions in sync">

Every time file content changes, increment `getScriptVersion` for that file. The language service uses version strings to decide whether to re-parse or re-type-check.

</Step>

<Step title="Dispose when done">

Call `languageService.dispose()` to release resources when you no longer need the instance.

</Step>

</Steps>

## `createLanguageService`

```typescript
function createLanguageService(
    host: LanguageServiceHost,
    documentRegistry?: DocumentRegistry,
    syntaxOnlyOrLanguageServiceMode?: boolean | LanguageServiceMode,
): LanguageService
```

Creates a language service instance backed by the given host.

<ResponseField name="host" type="LanguageServiceHost" required>
  The host that provides file content and compiler settings. See [LanguageServiceHost](#languageservicehost) below.
</ResponseField>

<ResponseField name="documentRegistry" type="DocumentRegistry">
  An optional shared registry for reusing `SourceFile` objects across multiple language service instances. If omitted, a private registry is created. See [DocumentRegistry](#documentregistry).
</ResponseField>

<ResponseField name="syntaxOnlyOrLanguageServiceMode" type="boolean | LanguageServiceMode">
  Pass `true` or `LanguageServiceMode.Syntactic` to skip type checking and only provide syntax-based features. Defaults to `LanguageServiceMode.Semantic`.
</ResponseField>

## `LanguageServiceHost`

The host interface your code must implement to drive the language service.

```typescript
interface LanguageServiceHost {
    // Required
    getCompilationSettings(): CompilerOptions;
    getScriptFileNames(): string[];
    getScriptVersion(fileName: string): string;
    getScriptSnapshot(fileName: string): IScriptSnapshot | undefined;
    getCurrentDirectory(): string;
    getDefaultLibFileName(options: CompilerOptions): string;
    readFile(path: string, encoding?: string): string | undefined;
    fileExists(path: string): boolean;

    // Optional — recommended
    getNewLine?(): string;
    useCaseSensitiveFileNames?(): boolean;
    readDirectory?(path: string, extensions?: readonly string[], exclude?: readonly string[], include?: readonly string[], depth?: number): string[];
    getDirectories?(directoryName: string): string[];

    // Optional — cancellation
    getCancellationToken?(): HostCancellationToken;

    // Optional — logging
    log?(s: string): void;
    trace?(s: string): void;
    error?(s: string): void;

    // Optional — custom module resolution
    resolveModuleNameLiterals?(...): readonly ResolvedModuleWithFailedLookupLocations[];
    resolveTypeReferenceDirectiveReferences?<T>(...): readonly ResolvedTypeReferenceDirectiveWithFailedLookupLocations[];

    // Optional — custom transformers
    getCustomTransformers?(): CustomTransformers | undefined;
}
```

## `IScriptSnapshot`

An immutable view of a file's text at a point in time.

```typescript
interface IScriptSnapshot {
    getText(start: number, end: number): string;
    getLength(): number;
    getChangeRange(oldSnapshot: IScriptSnapshot): TextChangeRange | undefined;
    dispose?(): void;
}
```

Create one from a plain string:

```typescript
const snapshot = ts.ScriptSnapshot.fromString(fileContents);
```

## `DocumentRegistry`

When you run multiple language service instances against overlapping sets of files (for example, one per open project in an editor), a shared `DocumentRegistry` lets them reuse parsed `SourceFile` objects instead of each parsing every file independently.

```typescript
function createDocumentRegistry(
    useCaseSensitiveFileNames?: boolean,
    currentDirectory?: string,
    jsDocParsingMode?: JSDocParsingMode,
): DocumentRegistry
```

Pass the registry to every `createLanguageService` call that should share it:

```typescript
const registry = ts.createDocumentRegistry();
const ls1 = ts.createLanguageService(host1, registry);
const ls2 = ts.createLanguageService(host2, registry);
```

The `DocumentRegistry` interface exposes `acquireDocument`, `updateDocument`, and `releaseDocument` methods for explicit lifecycle management, but in most cases the language service manages these automatically.

```typescript
    Semantic,        // Full type checking (default)
    PartialSemantic, // Type checking without project references
    Syntactic,       // Syntax only, no type checking
}
```

## Example: minimal in-memory language service

```typescript
import * as ts from 'typescript';
import * as path from 'path';
import * as fs from 'fs';

// In-memory file store
const files = new Map<string, { content: string; version: number }>();
files.set('/project/index.ts', { content: 'const x: number = "hello";', version: 1 });

const servicesHost: ts.LanguageServiceHost = {
    getScriptFileNames: () => ['/project/index.ts'],
    getScriptVersion: (fileName) => {
        const file = files.get(fileName);
        return file ? String(file.version) : '0';
    },
    getScriptSnapshot: (fileName) => {
        const file = files.get(fileName);
        if (file) {
            return ts.ScriptSnapshot.fromString(file.content);
        }
        if (fs.existsSync(fileName)) {
            return ts.ScriptSnapshot.fromString(fs.readFileSync(fileName, 'utf-8'));
        }
        return undefined;
    },
    getCurrentDirectory: () => '/project',
    getCompilationSettings: () => ({
        target: ts.ScriptTarget.ES2020,
        module: ts.ModuleKind.CommonJS,
        strict: true,
    }),
    getDefaultLibFileName: (options) => ts.getDefaultLibFilePath(options),
    fileExists: ts.sys.fileExists,
    readFile: ts.sys.readFile,
    readDirectory: ts.sys.readDirectory,
    getDirectories: ts.sys.getDirectories,
};

const languageService = ts.createLanguageService(
    servicesHost,
    ts.createDocumentRegistry(),
);

// Get semantic diagnostics
const diagnostics = languageService.getSemanticDiagnostics('/project/index.ts');
for (const diag of diagnostics) {
    const message = ts.flattenDiagnosticMessageText(diag.messageText, '\n');
    console.log(`Error: ${message}`);
}

// Update a file
files.set('/project/index.ts', { content: 'const x: number = 42;', version: 2 });
const noErrors = languageService.getSemanticDiagnostics('/project/index.ts');
console.log('Errors after fix:', noErrors.length); // 0

languageService.dispose();
```

<Tip>
  When you update a file's content, increment its version so the language service knows to invalidate its cached parse and type information.
</Tip>
